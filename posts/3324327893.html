CLI-Anything与OpenCli体验记录

发表于2026-04-07|更新于2026-04-08|计算机
|浏览量:

省流：在体验中我使用的是有点冷门的qwen code，模型是qwen3.6-plus，支持可能不如claude，codex那么好，但效果也是相当惊艳的。

CLI-Anything解决的是Agent与软件的接口问题，他选择在已知源码的软件和Agent间套一层CLI壳（也就是我们常说的中间层，我认为这个思路是相当好的），而不是走以往的GUI（十分脆弱），或者官方API（不够泛用），他也构建了一套完整流程让Agent来针对特定软件构建对应CLI，但真实效果可能并不如样例那样惊艳，老实说我觉得噱头居多，实际封装出来的CLI大多停留于简单的文本操作，离真正好用的CLI还差了很远。总结来说，稳定和泛化有了，但能力太弱了。

OpenCli可以理解为极速爬虫，他是通过Chrome 扩展与浏览器建立连接，可以复用浏览器的登录状态，从而极简地获取支持的网页的状态，不需要配置任何API Key，还能够控制桌面 Electron 应用，甚至可以让ai自己控制自己执行任务。根本上我认为是给了一个便捷的供Agent查询网页信息和调度的接口，无论是自动化脚本还是供Agent使用都是有价值的。

CLI Anything

HKUDS/CLI-Anything: “CLI-Anything: Making ALL Software Agent-Native” – CLI-Hub: https://clianything.cc/

去年我玩过一个类似的，叫windows-use，与windows gui层接轨，声称能完成自己打开文档写文档，但实际十分脆弱，没走两步就不知道在干什么了。但用官方API又不够泛用。CLI Anything相当于提供了我们常说的中间层，如果c语言直接转为机器码困难，就加了一层汇编指令集。保证了泛用性和稳定性，我觉得是相当好的，但实际用起来还有很大优化空间。

构建所需软件CLI

官方并没有直接给Qwen Code的SKILL.md，我就改了改Claude Code的SKILL.md接进Qwen Code，尝试给Drawio构建了一个CLI，看转化出的代码老实说是比较单薄的，大多停留在简单的文本操作，实测在使用颜色上有bug，但也能正常使用生成流程图。

插件里的 HARNESS.md把CLI构建 定制为7 阶段自动化流水线，实现了把“软件适配给 Agent”这一要求的流程化。

所以理所当然的，他比较适合开源桌面软件，能进行详细结构分析，对闭源软件和复杂软件的构建就很可能会失败。

使用CLI

他自己维护了一个cli-anything-hub，存他转化好的各种CLI，所以只需要增加一个SKILL.md,这个skill就是让Agent去github上的cli-anything-hub查找什么软件适合完成该任务，然后pip install 对应的CLI。

我让他用drawio画一个乱序CPU流程图，从找hub配环境到完成绘制只用了3分钟，但画出来一言难尽,CLI本身绘画功能有限，也没有正常实现颜色等功能。

所以我觉得更像是一个方向指示性的项目，但离成熟地完成工作还比较远。一个噱头算是吧,方向确实很不错，模型再发展以及本身逻辑优化后可能真的会出现泛用成熟的。

OpenCLI

jackwener/opencli: Make Any Website & Tool Your CLI. A universal CLI Hub and AI-native runtime. Transform any website, Electron app, or local binary into a standardized command-line interface. Built for AI Agents to discover, learn, and execute tools seamlessly via a unified AGENT.md integration.

封装很好，配置很简单

  1. Install Browser Bridge Extension

OpenCLI connects to your browser through a lightweight Browser Bridge Chrome/Chromium extension + micro-daemon (zero config, auto-start).

  1. Go to the GitHub Releases page and download the latest opencli-extension.zip.

  2. Unzip the file and open chrome://extensions, enable Developer mode (top-right toggle).

  3. Click Load unpacked and select the unzipped folder.

  4. Install OpenCLI

Install via npm (recommended)

安装拓展和npm安装后就可以直接终端使用了

还可以npx skills add jackwener/opencli让Agent学会通过OpenCil查询网络信息。

也就是不需要API Key就能极速爬虫，可以把 OpenCLI 无缝集成到各种自动化流程中，支持的主流网站和接口有30+

甚至还能用来下bilibili视频（

十分适合用于各种自动化脚本，比如博客爬一下最近数据，即使不和Agent扯到一起也是个十分优秀的项目，当然Agent也能通过它进行主流网页的信息抓取，虽然感觉Agent可能更多是通过搜索引擎API？

总之是个潜力和应用场景都很大的优秀项目。但感觉不必太用来炒Agent的概念。
